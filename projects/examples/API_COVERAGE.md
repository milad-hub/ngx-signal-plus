# Examples API Coverage

The examples application is organized by workflow rather than one screen per export. This map links each public API group to its interactive section, with the handful of lower-level classes covered by the full reference instead.

| Public API group                | Interactive coverage                                                                                                                                            | Reference-only APIs                                                                   |
| ------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------- |
| Signal creation and enhancement | Counter (`sp`), editor, cart, native-signal `enhance`, `spToggle`, and bounded `spCounter`                                                                       | `spSignalBuilder` (used internally by `sp`)                                            |
| Presets and validators          | `spForm.text` / `spForm.email` inputs, `spPresets.searchField`, and `spValidators.number.range` on the score signal (§10 Presets)                                | Remaining `spPresets` variants (`counter`, `toggle`, `formInput`, `persistentToggle`)   |
| Validation and schemas          | Form validation, `spSchemaValidator`, `spSchema`, and `spSchemaWithErrors` with live error output (§11)                                                          | —                                                                                       |
| Collections and async state     | Todo collection, local async state, and `spStorageManager` save/load snapshot (§11)                                                                              | `spHistoryManager` (drives `.withHistory()` in every history demo)                      |
| Forms                           | Grouped name, email, and age controls with `spFormGroup`                                                                                                          | `spSignalPlusComponent`, `spSignalPlusService`                                          |
| Queries                         | Query, mutation, infinite query, and dependent query                                                                                                              | `createQuery`, `createMutation`, `createInfiniteQuery`, `QueryClient`, `getGlobalQueryClient`, `setGlobalQueryClient` (the `sp*` wrappers demonstrated are thin aliases) |
| Transactions and batching       | Form transaction, cart batch, and live `spIsInTransaction` / `spIsInBatch` / `spGetModifiedSignals` introspection (§11)                                          | `spIsTransactionActive`                                                                 |
| Middleware and diagnostics      | Logger middleware, `spGetMiddlewareCount`, debug state, monitoring metrics, and effect controller                                                                 | `spAnalyticsMiddleware`, `spClearMiddleware`, `spRemoveMiddleware`                      |
| Operators and composition       | Operator pipeline (§9): `spMap`, `spFilter`, `spDebounceTime`, `spDistinctUntilChanged`, `spThrottleTime`, `spDelay`, `spSkip`, `spTake`, `spMerge`, `spCombineLatest`; plus `spCombine`, `spAll`, `spAny`, `spComputed` | —                                                                                       |
| Error utilities                 | `spCreateError` and `formatSpError` render a formatted `SpError` when the bounded counter breaks (§11); validation failures also surface in the form example      | `SP_ERRORS` table (source of the codes shown)                                           |

For method signatures, error codes, and advanced configuration, use the [full API reference](../signal-plus/docs/API.md).
